# 大模型批处理

## 1. 静态批处理

当批量传入token序列的时候，在批处理请求中所以prompt完成后完成响应，批处理的大小在推理完成之前保持不变。

这样实践起来十分简单，但是会导致生成长度最长的序列阻塞其他的prompt

## 2. 动态批处理

动态批处理是指允许将一个或者多个推理请求合并为单个批次的请求，其有点类似班车，在原地等待一段时间，尝试将若干batch请求合并为一个大batch，从而提高吞吐量。

## 3. 连续批处理

指的是发现当前已经处理完了一个请求，那么尝试将新的请求插入到当前处理中的请求中，从而提高吞吐量，这点需要结合pagedAttention进行实现
