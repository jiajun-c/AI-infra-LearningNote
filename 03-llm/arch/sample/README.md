# 大模型采样策略

模型输出的每个token的logits是未归一化的的，采样策略决定如何从中选出下一个token

## 1. 贪心解码

每一步直接选择概率最大的token

Logits -> softmax -> [0.05, 0.7, 0.15, 01]，直接选择0.7的那个

优点：确定性输出，速度快
缺点：容易重复，生硬，缺乏多样性，可能陷入局部的最优

## 2. Tempature 

可以减少概率分布的尖锐，平坦程度

P(x_i) = exp(z_i/T) / `sum(exp(z_j/T))

T < 1 的时候是低温，输出更加稳定，更加适合进行代码生成

T > 1 的时候是高温，输出更加随机，更加有创意

## 3. Top-K 采样

只保留概率最高的K个token，其余概率都置于0,在K个中重新归一化采样

K=3 时

原始概率：[0.4, 0.25, 0.14, 0.10, 0.05, 0.03, 0.02]

只保留三个，然后重新进行归一化

## 4. Top-p 采样

将概率从大到小进行排列，累加到概率综合刚好超过P的位置截断，在这个动态集合中采样

## 5. Beam search

维持beam width 大小的路径，例如第一步选了两个，两个派生出8个，再从8个里选两个概率最高，按照这个方式进行类推
