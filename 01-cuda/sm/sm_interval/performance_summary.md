# SM 放置策略性能对比总结

## 测试环境

- **GPU**: NVIDIA H100 80GB HBM3 (132 SMs)
- **Active SMs**: 66 (50% 利用率)
- **Threads/block**: 256
- **Warmup iters**: 10
- **Test iters**: 20

## 两种策略

| 策略 | SM 分布 | L2 Partition 利用 |
|------|---------|------------------|
| **Sequential** | SM 0-65 (连续) | 前 8 个 Partition |
| **Interleaved** | SM 0,2,4,...,130 (偶数) | 全部 16 个 Partition |

## 关键发现

### 1. 甜蜜点：2048 × 4096 (32 MB)

```
矩阵大小：32 MB (占 L2 容量的 64%)
Interleaved 优势：+19.14% (1147.76 vs 963.41 GB/s)
```

**原因分析**：
- Sequential (SM 0-65) 只使用前 8 个 L2 Partition，带宽利用率 ~50%
- Interleaved (SM 0,2,4,...,130) 分散到全部 16 个 L2 Partition，带宽利用率 ~100%
- 32 MB 数据部分放入 L2，L2 带宽成为瓶颈

### 2. 小矩阵区域 (<10 MB)：Sequential 更优

```
典型：512 × 1024 (2 MB)
Sequential 优势：+5.33%
```

**原因分析**：
- 整个矩阵可放入 L2
- Sequential 的连续 SM 分布有更好的数据局部性
- L1 预取更有效

### 3. 大矩阵区域 (>50 MB)：差异消失

```
典型：8192 × 8192 (256 MB)
差异：-1.48% (可忽略)
```

**原因分析**：
- L2 几乎全部 miss，直接从 HBM 读取
- 瓶颈在 HBM 带宽 (3.35 TB/s)
- SM 分布对 HBM 访问模式影响小

## 实际建议

### 何时使用 Interleaved

| 条件 | 推荐策略 | 预期收益 |
|------|---------|---------|
| 矩阵大小 ≈ L2/2 (16-48 MB) | **Interleaved** | +15~20% |
| 矩阵大小 ≈ L2/4 (8-16 MB) | **Interleaved** | +2~5% |
| 多 Kernel 并发 | **Interleaved** | 减少 L2 争用 |
| 不规则访问模式 | **Interleaved** | 分散热点 |

### 何时使用 Sequential

| 条件 | 推荐策略 | 预期收益 |
|------|---------|---------|
| 矩阵大小 << L2 (<5 MB) | **Sequential** | +3~10% |
| 相邻数据复用高 | **Sequential** | L1 预取更有效 |
| 小 batch 推理 | **Sequential** | 简单，易调试 |

### 何时差异可忽略

| 条件 | 策略选择 |
|------|---------|
| 矩阵大小 >> L2 (>100 MB) | 任意 (HBM 瓶颈) |
| 矩阵大小 ≈ L2 (50 MB) | 任意 (L2 几乎 miss) |

## H100 L2 架构洞察

```
H100 L2 (50 MB) = 16 Partitions × ~3.1 MB

关键设计原则：
1. L2 是 Partition 化的，不是均匀一块
2. SM 放置决定 L2 Partition 利用率
3. 中等矩阵大小时，Partition 分布影响最大化
4. 极小或极大矩阵时，差异不显著
```
