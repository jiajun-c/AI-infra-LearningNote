# IO_URING

## 1. libaio 的问题

尽管说libaio解决了并发调用的问题，但是在面临百万级的IO请求的时候，在每次提交任务和收割任务的时候都会陷入一次内核态的调用，从而影响性能

## 2. io_uring 如何解决

io_uring 彻底掀翻了桌子。它通过 mmap（内存映射）在用户空间和内核空间之间建立了一块共享的内存区域。

- SQ(提交队列): 用户态把要提交的事情打包到这里
- CQ(完成队列): 内核把完成的结果提交到这里

革命性的改变在于： 因为内存是共享的，应用程序往 SQ 里塞任务，或者从 CQ 里拿结果，变成了纯粹的用户态内存读写（改几个指针），彻底消灭了系统调用！
